# Tech News Bot

Телеграм-бот, который читает технологические Telegram-каналы и помогает понять, **о чём они пишут, насколько это интересно аудитории и какие темы сейчас в тренде**. Вся аналитика приходит прямо в чат с ботом — никаких таблиц и ручного подсчёта.

## 1. Стек

Что у бота «под капотом»:

| Слой | Технология | Зачем нужна |
|---|---|---|
| Интерфейс бота | **Aiogram 3** | Принимает команды в Telegram, рисует кнопки и пошаговые диалоги |
| Чтение каналов | **Telethon** | Заходит в Telegram как обычный пользователь и читает посты каналов |
| База данных | **PostgreSQL** | Хранит каналы, посты и результаты анализа |
| LLM | **OpenRouter** (OpenAI-совместимый API) | Определяет темы постов, описывает стиль и аудиторию канала |
| Векторный поиск | **ChromaDB** + **Ollama** | Сравнивает посты «по смыслу», а не по словам (находит похожие и дубликаты) |
| Кэш и состояние диалогов | **Redis** | Запоминает, на каком шаге диалога находится пользователь, и ускоряет повторные запросы |
| Планировщик | **APScheduler** | Присылает дайджесты автоматически по расписанию |
| Запуск | **Docker / Docker Compose** | Поднимает бота и все сервисы одной командой |

## 2. Для чего нужен бот

Представьте, что вы следите за десятком технологических каналов и хотите быстро понять:

- **О чём вообще канал** и стоит ли на него подписываться?
- **Какие посты «зашли»** аудитории за неделю, а какие — нет?
- **Какой из двух каналов** интереснее и активнее?
- **Какие темы набирают популярность**, а какие угасают?

Раньше для этого пришлось бы вручную листать ленту и считать лайки. Бот делает это за вас: собирает посты, считает метрики (просмотры, реакции, комментарии), а искусственный интеллект подсказывает темы, стиль и портрет аудитории. Результат — короткая понятная сводка прямо в Telegram.

## 3. Функции бота

У бота четыре основные команды. Ниже — что подать «на вход» и что вы получите «на выходе».

### `/analyze` — анализ одного канала

**Что делает:** разбирает один канал «по косточкам».

**Что нужно ввести:**
- Ссылку или имя канала, например `@durov` или `https://t.me/durov`.
- Можно сразу `/analyze @durov`, а можно просто `/analyze` — бот сам спросит канал.

**Что получите на выходе:**
- Основные **темы** канала.
- Описание **аудитории** (для кого канал) и **стиля** подачи.
- **Метрики**: средние просмотры, реакции, комментарии.
- **Топ постов** канала со ссылками.

---

### `/digest` — дайджест лучших постов

**Что делает:** собирает самые удачные посты из нескольких каналов за выбранный период — как «лучшее за неделю».

**Что нужно ввести** (бот ведёт по шагам с кнопками):
- **Список каналов**, которые интересны.
- **Период** — например, за день, неделю или месяц.
- При желании — **ключевые слова**, чтобы оставить только посты на нужную тему.
- При желании — **расписание**, чтобы дайджест приходил автоматически (например, каждый понедельник утром).

**Что получите на выходе:**
- Подборку **лучших постов** со ссылками и метриками.
- Краткую сводку: какие **темы** преобладали и насколько активна была аудитория.
- Если включили расписание — такие дайджесты будут приходить сами.

---

### `/compare` — сравнение двух каналов

**Что делает:** ставит два канала рядом и показывает, кто сильнее.

**Что нужно ввести:**
- **Два канала** для сравнения (например, `@channel1` и `@channel2`).
- **Период**, за который сравниваем.

**Что получите на выходе:**
- **Метрики бок о бок**: просмотры, реакции, вовлечённость (ER) обоих каналов.
- **Общие темы** — о чём пишут оба, и **уникальные темы** — чем каждый отличается.
- Сравнение **стиля** подачи.

---

### `/trends` — тренд-радар тем

**Что делает:** показывает, какие темы сейчас **на подъёме**, а какие **угасают**. Это чистая аналитика динамики — удобно, чтобы поймать «волну».

**Что нужно ввести:**
- **Список каналов** для наблюдения.
- **Период**, за который смотрим динамику.

**Что получите на выходе:**
- **Растущие темы** — о чём стали писать чаще.
- **Угасающие темы** — что теряет популярность.
- **Новые темы**, которых раньше не было.
- **Самый виральный пост** за период.

> Дополнительно: команда `/cancel` отменяет текущий пошаговый диалог в любой момент.

## 4. Как запустить

Бот рассчитан на запуск через **Docker** — так одной командой поднимаются и сам бот, и все нужные сервисы (база данных, кэш и т. д.).

### Шаг 1. Подготовьте настройки

В файле `.env` (в корне проекта) укажите ключи доступа. Пример полей есть в `.env.example`:

- `BOT_TOKEN` — токен бота от [@BotFather](https://t.me/BotFather).
- `TG_API_ID`, `TG_API_HASH`, `TG_PHONE` — данные для чтения каналов через Telethon (получаются на [my.telegram.org](https://my.telegram.org)).
- `LLM_API_KEY`, `LLM_MODEL`, `LLM_BASE_URL` — доступ к LLM (OpenAI-совместимый API, напр. OpenRouter).
- `EMBED_MODEL` — название модели эмбеддингов в Ollama. Эту модель нужно один раз загрузить в Ollama — см. Шаг 4.
- Остальные адреса баз данных и сервисов уже прописаны в `docker-compose.yml`.

### Шаг 2. Авторизуйте чтение каналов (один раз)

Чтобы бот мог читать каналы, нужно один раз войти в Telegram-аккаунт — Telegram пришлёт **код подтверждения**. Этот вход делается **в обычной консоли** (не в Docker), потому что код надо ввести вручную:

```powershell
python main.py
```

После ввода кода создаётся файл сессии в папке `sessions/`. **Дальше код больше не понадобится** — Docker будет использовать готовую сессию.

> Если код «не приходит» при повторном запуске — скорее всего, вход уже выполнен и сессия готова. Просто запускайте через Docker (Шаг 3).

### Шаг 3. Запустите всё через Docker

```powershell
docker compose up -d
```

Эта команда поднимает бота и все сервисы (PostgreSQL, Redis, ChromaDB, Ollama). После старта бот готов принимать команды в Telegram.

Посмотреть, что бот работает без ошибок:

```powershell
docker compose logs -f bot
```

Остановить всё:

```powershell
docker compose down
```

### Шаг 4. Загрузите модель эмбеддингов в Ollama (один раз)

Ollama поднимается в Docker **пустой** — нужную модель эмбеддингов в неё надо загрузить один раз. Сделайте это после `docker compose up -d`, указав ту же модель, что прописана в `EMBED_MODEL`:

```powershell
docker exec -it ollama ollama pull qwen3-embedding
```

Модель сохраняется в volume `ollama_models`, поэтому при перезапусках качать заново не нужно. Проверить, что модель на месте:

```powershell
docker exec -it ollama ollama list
```

### Подсказки

- Логи бота также пишутся в файл `logs/bot.log`.
- Если запустить `python main.py` **без** поднятых сервисов — бот не сможет подключиться к Redis и базе и завершится с ошибкой. Поэтому полноценный запуск — только через Docker.
